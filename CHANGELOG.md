### v3.0 — Agent方法论重设计（2026-04-24）

**核心改变**：把agent从"风格各异的评论员"升级为"方法论不同的AI分析系统"

| 改动 | 效果 |
|------|------|
| 月影姐：从"玄学随机"改为"反叙事猎手" | 不再每次都押冷门，不可预测 |
| 冰狗：必须给P(主胜)/P(平)/P(客胜)三个数字 | 发言更可争辩，有明确论据 |
| 赌狗：区分公众盘/职业盘，给出被错误定价的一边 | 不再只是"盘口低就梭哈" |
| 碎碎念：必须点名球员+给出可观察信号来源 | 不再空洞的"心理分析" |

---

### v3.1 — 方法论碰撞+数据修复（2026-04-24）

**BUG修复**
- 删除25个空缓存（旧缓存写入了空数组`[]`，被误判为有效数据，导致fetchTeamForm/fetchH2H永远返回空）

**新增**
- `AGENT_METHOD` 映射：每个agent的方法论核心主张
- `METHOD_CLASH` 碰撞矩阵：20组对战配对各有专属碰撞要点
- `buildDebateDirective()` 重写：注入方法论碰撞 + 自身真实数据
- Reaction阶段改为双向互怼（A质疑B + B反击A）
- 第三agent插嘴：round 2后65%概率触发旁观者视角
- AGENT_PERSONAS全面重写：加入AI超能力（Poisson/舆情/盘口监测/行为语言/历史匹配）

**调整**
- 阅读时间：每字80ms + 3000ms基础，下限7s，上限15s
- 发言面板 max-height 155px → 300px

### v4.8 — LLM后端迁移（2026-04-29）

**背景**：Moonshot moonshot-v1-8k出现error 42限速，并发导致卡死

| 改动 | 说明 |
|------|------|
| LLM后端迁移 | Moonshot → 火山方舟 DeepSeek-v3.2 |
| 双key轮询 | VOLC_API_KEY_1/2交替，配额翻倍 |
| 初判完全串行 | for...of循环替代Promise.allSettled，根治并发卡死 |
| 流式输出 | stream:true，TTFT ~1s |
| speaking_start事件 | 首字节时立即触发3D发言动画（1s vs 9s） |
| JSON mode | response_format替代tool_choice（速度+1-2s） |
| PUBLIC_PROMPT精简 | 350→100token，历史slice从14→8条 |
| 终投差异化 | buildVoteDirective()：五人独家视角，防重复发言 |

**根因修复记录**
- 卡死问题：并发触发速率限制，display loop整体阻塞 → 改串行彻底解决
- 重复终投：所有人被问同样3个问题 → 每人专属任务（冰狗更新概率/赌狗判断职业盘/老球迷补历史/碎碎念分析辩论心理/月影姐更新叙事强度）
