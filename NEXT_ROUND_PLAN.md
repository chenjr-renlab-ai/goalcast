### 4.2 用户流程失败案例

**失败案例 FA-1：新用户第5秒流失**
- 用户通过引导弹窗后，看到的是一个**暗色 3D 场景，6 个小人站着不动，背景是巨大镜像文字**
- 没有任何正在发生的事情，没有"试看"功能，没有"上一场议会长这样"的预览
- 用户无法判断"我接下来要等多久"和"这个东西到底在做什么"
- **结论**：空闲状态对首次用户来说是死胡同，5 秒内没有"哇"的时刻

**失败案例 FA-2：长等待没有进度感**
- 点击召开议会后，5 个 agent 依次发言，每人等待约 8s，总计 40s 初判期
- 期间只有 3D agent 的 thinking 动画和 speaking_start 触发的摄像机切换
- 没有"第 2/5 位专家正在分析"的进度指示器，用户不知道还要等多久
- **结论**：感知等待比实际等待更痛苦；需要进度可视化

**失败案例 FA-3：trust = 0 的首次体验**
- 新用户无法看到"这个 AI 之前预测对过多少次"
- agent 准确率徽章只有在议会结束后的历史记录里才能看到
- 没有任何社会证明（"已有 XXX 场预测"、"近期命中率 XX%"）
- **结论**：无信任建立手段，用户无法判断预测价值

**失败案例 FA-4：分享路径太深**
- 分享战报按钮在结果页最底部，需要完成整场 8+ 分钟的议会才能到达
- 分享图的存在感对于"还没开始"的用户完全不可见
- **结论**：如果用户中途退出（40s 初判期），分享路径永远触达不到

**失败案例 FA-5：移动端失去核心卖点**
- 375px 宽度下 3D 场景完全消失，只剩 agent 卡片
- 3D 场景是产品最大的视觉差异化，移动端用户看不到
- 移动端 3D 缺失 + 40s 等待 = 极高流失率
- **结论**：移动端需要独立的体验设计方案

**失败案例 FA-6：品牌定位模糊**
- topbar 写"FIFA · WORLD CUP 2026 · PREDICTOR"，但预测的是英超联赛
- 产品英文名 ORACLE COUNCIL，中文名"预言者议会"，两者在 UI 中混用
- 目标用户（中国英超球迷）看到 FIFA 世界杯副标题会产生认知偏差
- **结论**：品牌副标题需要和实际产品内容对齐

## 五、产品定位重新梳理（v4.9 方向）

### 5.1 产品的核心价值主张

不是"AI 预测工具"，而是**"把足球分析变成综艺节目"**。

核心差异：
- 普通 AI 预测 → 给你一个数字
- 预言者议会 → 给你 6 个有个性、有方法论、会吵架的 AI 角色，用不同视角解读同一场比赛

用一句话说：**"比足球本身更好看的，是关于足球的辩论。"**

### 5.2 目标用户画像

| 用户类型 | 描述 | 核心需求 | 当前满足度 |
|---------|------|---------|-----------|
| **主力：泛球迷** | 20-35岁，关注英超，不是深度分析派 | 在比赛前"了解个大概"，看点娱乐内容 | ★★★☆☆ |
| **潜力：AI 爱好者** | 对 AI 技术感兴趣，想看 AI"实战表现" | 看 AI 之间的意见分歧，判断哪个 AI 更聪明 | ★★☆☆☆ |
| **边缘：休闲博彩用户** | 偶尔买竞彩，想要"参考意见" | 赔率分析、盘口解读 | ★★☆☆☆ |
| **排除：专业分析师** | 需要真实量化数据和模型输出 | — | 不是目标 |

**核心用户场景**：比赛前 30 分钟，用户打开 APP，看 AI 议会预测一下，顺便看看今晚的看点是什么，然后开始看球。

### 5.3 营销切入点（按传播力排序）

1. **"看 AI 吵架"** — 6 个 AI 用完全不同的方法互怼，赌狗说数据帝是书呆子，数据帝说赌狗是赌徒。这本身就是内容。视频剪辑发小红书/抖音有传播潜力。

2. **"你赢了 AI 议会吗？"** — 用户预测 vs AI 集体预测，形成博弈感。结束后分享"我押平局，AI 全押主队赢，最后我对了"——有强烈的炫耀动机。

3. **"每场比赛的6种解读"** — 教育属性：让不太懂球的人也能通过赌狗/历史/心理视角"理解"这场比赛。让懂球的人发现角度盲区。

4. **"AI 准确率 PK"** — 长期运营价值：哪个 AI 角色预测最准？冰狗还是赌狗？用户可以关注和追踪。

## 八、世界杯转型战略（2026 FIFA World Cup）

> **背景**：产品定位本就是蹭世界杯热度，英超数据是过渡期跑通产品逻辑的临时方案。
> WC 2026 开赛时间：2026年6月11日（约6周后），赛程至7月19日。
> **最重要背景**：中国队时隔24年再度晋级世界杯（2002年后首次），这对中国球迷是历史级事件，是产品最大的流量窗口。

### 8.1 赛事参数差异（EPL vs WC 2026）

| 维度 | 当前（英超） | 目标（WC 2026） |
|------|------------|--------------|
| 队伍数量 | 20支俱乐部 | 48支国家队 |
| 赛制 | 联赛滚动赛季 | 小组赛(12组×4) → 32强 → 16强 → 8/4/2/决赛 |
| 情感烈度 | 中 | **极高**（国家荣誉、4年一届） |
| 中国受众关注度 | 中（英超球迷） | **极高**（全民关注，中国队参赛） |
| 场次密度 | 每轮10场 | 小组赛每天3-4场，赛程紧凑 |
| 赔率市场 | 相对稳定 | 波动大，爆冷高频（弱队 upset） |
| 主客场逻辑 | 明确主客 | **中立场地**（北美三国承办）|
| 传播周期 | 全年持续 | 集中爆发6周 |

### 8.2 数据源适配方案

| Agent | 当前数据源 | WC 适配方案 |
|-------|----------|-----------|
| 冰狗（统计） | football-data.org 英超进失球 | football-data.org `/v4/competitions/WC` + FIFA 排名 + 世预赛数据 |
| 赌狗（盘口） | the-odds-api EPL 赔率 | the-odds-api WC 赔率（已支持，sport=soccer_fifa_world_cup） |
| 老球迷（历史） | football-data.org H2H | 国家队历史交锋（含往届世界杯战绩）|
| 碎碎念（心理） | FPL 球员伤情新闻 | **FPL无WC数据** → 改用 SofaScore/TransferMarkt 国家队名单 + 赛前新闻 |
| 月影姐（舆情） | LLM 模拟舆情 | 同上（已是模拟，只需更新叙事背景） |
| 议长 | 综合所有 | 同上 |

**技术难点**：FPL 仅覆盖英超球员，WC 需要替代球员状态数据源。
- 短期方案：无 FPL 时 psych agent 改用"赛前新闻文本 + 教练发布会分析"作为数据
- 长期方案：接入 SofaScore 非官方 API 或爬取国家队伤病名单

### 8.3 内容策略调整

**角色指令优化（WC 专属语境）：**

- **冰狗**：把"英超近5场"换成"本届世预赛进失球统计 + FIFA排名变化趋势"。加入"冷门概率"——WC历史上低排名队伍爆冷的频率。
- **赌狗**：WC 盘口特点：开赛前几天赔率剧烈波动，"公众盘"偏向大队，"职业盘"经常悄悄支持爆冷。这比英超盘口更戏剧化。
- **老球迷**：WC 历史比英超更有故事性（1950年乌拉圭奇迹、2002年韩国奇迹、2010年西班牙首冠）。国家队 H2H 包含更多"宿命感"叙事。
- **碎碎念**：球员赛前压力比俱乐部赛更大（国家荣誉 vs 职业合同）。主力球员的伤病处理方式、替补阵容变化更有心理分析价值。
- **月影姐**：WC 的舆情叙事烈度是英超的10倍。"X国必赢"、"爆冷预言"在社交媒体形成巨大泡沫。反叙事机会更多。

**中国队特殊处理**：

## 九、议事厅前端界面深度优化方案

> 基于 gstack 审查结论，针对 3D 场景和整体 UI 的系统性优化设计。
> 原则：不改变产品功能逻辑，只提升视觉表现力和用户感知质量。

### 9.1 3D 场景核心问题与解法

**问题一：背景看板文字统治全屏**

现状：Three.js 后墙面板渲染"ORACLE COUNCIL · AI PREDICTION"大字，因为墙在 agent 背后，文字镜像反转，在 1280px 宽度下占据屏幕 60% 面积，agent 只剩下方 20%。

解法：
```
A. 短期（低难度）：把背景板文字 opacity 降至 0.15，scale 缩小至 40%
B. 中期（推荐）：背景板改为"赛事动态可视化"—— 用抽象的动态粒子/光波纹替代可读文字
C. 长期（WC版）：背景板显示对战国旗的艺术化渲染（半透明、模糊处理）
```

**问题二：Agent 模型相对屏幕太小**

现状：RADIUS = 5.5（估算），摄像机在距离 ~18 的位置，agent 高度约占屏幕 15%。

解法：
```
A. 减小摄像机默认距离（camState.pos z: 18 → 13）
B. 缩小 RADIUS（5.5 → 4.0）让 agent 更集中
C. 发言时摄像机拉近更多（当前 z: 9 → 6-7）
D. 给每个 agent 的地台（hex platform）增加光柱向上发散，增大视觉存在感
```

**问题三：空闲状态无生命力**

现状：无 session 时，6 个小人站着呼吸，背景静止，看板静止，和截图无区别。

解法：
```
A. 空闲时摄像机缓慢绕圆心旋转（orbitPaused=false 时已有，增加旋转幅度）
B. Agent 随机做"转头看向其他人"的动作（随机选 2 个 agent 互相 rotation.y 对视）
C. 中央地台发散柔和的能量光圈（3-5 个 TorusGeometry，透明，缓慢向外扩散消失）
D. 顶部照明出现"追光灯"效果（RectAreaLight 缓慢扫描场地）
```

**问题四：发言时沉浸感不足**

现状：发言时摄像机切到该 agent，其余人变暗，有光晕和粒子爆发，但整体氛围仍平淡。

解法：
```
A. 发言时地台光束颜色加深，高度增加（beam opacity: 0.16 → 0.28）
B. 发言 agent 的外圈增加"音波"效果（TorusGeometry 从 0.7 → 1.2 逐渐扩散消失）
C. 当 catchphrase 触发时，在场景中央短暂出现金色粒子爆炸
D. pivot 转向时，对应 agent 的颜色从原来的颜色切换到新立场颜色（蓝/红/黄切换）
```

### 9.2 Agent 卡片区（左右两列）优化

**现状问题**：空闲状态卡片只显示名字、职称和"各场预测中..."占位符，6 张空卡片传递零信息。

**优化方案**：

```
卡片区分三个状态显示不同内容：

状态1 - 空闲（无 session）：
  ┌─────────────────┐
  │ 🕶️ Dr.冰狗      │
  │ 数据帝           │
  │ ────────────── │
  │ 方法：Poisson 模拟│
  │ 擅长：进失球概率  │
  │ 盲点：不信心理   │
  │ 近5场命中率 ██%  │
  └─────────────────┘

状态2 - 进行中（session active）：
  ┌─────────────────┐
  │ 🕶️ Dr.冰狗      │  ← 发言时高亮边框
  │ ────────────── │
  │ 当前立场：主队赢  │
  │ 置信度：68%      │
  │ 关键因素：xG差距  │
  └─────────────────┘

状态3 - 结束后：
  ┌─────────────────┐
  │ 🕶️ Dr.冰狗      │
  │ ✅ 终投：主队赢  │
  │ "主胜概率68%…"  │  ← catchphrase
  │ 本场命中：✅/❌  │
  └─────────────────┘
```

**实现要点**：
- 空闲状态：从 `AGENTS` 配置中读取固定的方法标签和盲点文字，填入卡片
- 进行中：从 `blackboard.agentStances` 动态读取立场
- 精准度：从 `/api/memory/profiles` 接口获取命中率数据

### 9.3 整体 UI 配色与视觉调整

**当前问题**：

### 9.4 移动端（375px）专项优化

**当前状态**：3D 场景完全隐藏，只有 agent 卡片 + 控制条 + LIVE 条。核心视觉卖点丢失。

**解决方案（两步走）**：

```
第一步（低成本，可立即做）：
- 移动端显示一个"3D 场景缩略预览图"（静态截图或低帧 canvas）
- 在 session 进行时改为"当前发言者全屏大头照模式"
  └── canvas 只渲染当前发言的 agent 特写（camera 极近，背景虚化）

第二步（中成本，WC 上线前）：
- 移动端保留 3D canvas，但高度限制为 screen 的 35%
- 只渲染 3 个 agent（当前立场分歧最大的两个 + 议长）
- 发言时切全屏 canvas（gesture swipe down 可收起）
```
