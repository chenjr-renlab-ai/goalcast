### 截图观察

**顶栏**：54px高，能塞下"@2.5 VS @2.8"和队名，算是信息密集，但字体小。阶段指示器（开场/初判/对线/终投）功能存在，但视觉权重不够。

**概率条**：74px，三段蓝/深绿/红，都显示"—"。深绿段和黑背景区分度极低（几乎看不出边界）。没有动态感，用户不知道这条会发生什么变化。

**Agent左列**：148px宽，只能显示小图标+名字+职称。极度信息稀缺——用户完全无法判断这个角色有没有说服力、历史准确率如何、当前押什么。

**3D场景**：6个Roblox风格人形，带LED背景屏和球场地板。场景本身花了很多工程量，但从第一视角来看：空旷，暗，卡通，游戏感。和"AI预测分析"的期望完全不匹配。

## 二、核心矛盾诊断

### 信息层级完全倒置

**现在的空间分配逻辑**：
```
高价值内容（Agent分析、预测数据、方法论）→ 放在最小的区域（148px列）
低价值内容（3D空气效果）→ 占据77%宽度
关键数据（赔率、近况）→ 埋在26px滚动条里
```

**应该是**：
```
1号位（最大空间）→ 当前预测走势（概率的动态变化）
2号位（次大空间）→ 发言内容 + 数据来源
3号位（次要）→ Agent身份面板（谁在说，历史多准）
背景层 → 3D氛围（装饰，不是主角）
```

### 两个失败的用户体验

**失败1：新用户不知道在干什么**

用户加载页面后看到：
- 6个卡通人物站在绿色圆圈里什么都不做
- 左右有一排名字卡
- 底部有"召开议会"按钮
- 无解释、无引导、无预期设置

访客的认知路径："这是一个游戏吗？" → 点击"召开议会" → 等待 → 文字出现 → "哦，是AI聊天" → 继续看 → "他们说的是什么依据？" → 找不到答案 → 失去信任

**失败2：看了也不信**

试用用户原话："看乐子可以，但是不会信，除非你摆个数据or告诉我你怎么得到的"

这句话指向的信任缺口：
1. **没有可见的数据依据** — 代码里有真实API数据，但用户界面上看不到它们被引用
2. **没有历史验证** — Agent历史准确率系统已经实现，但界面上完全不显示
3. **没有方法论说明** — 每个Agent有严格的方法论定义，用户完全不知道
4. **3D玩具角色破坏可信度** — Roblox风格的人形让预测看起来像一个游戏而非分析

## 三、学术性 vs 趣味性的平衡分析

### 失败案例：走向两个极端

**极端A（过学术）**：显示置信区间、p值、样本量 → 没人看完，连球迷都跑了

**极端B（过娱乐）**：纯卡通 + 弹幕梗词 + 角色喊话 → 看了几场就腻了，且完全不可信

### 当前产品在哪里

当前设计在走**娱乐路线**（卡通人形、弹幕风语气、深绿电竞配色），但**娱乐质量不够高**（角色不生动、动效平淡、节奏感弱），同时**可信度为零**（没有任何数据可见性）。

结果：两边都没到达——娱乐性不够让人上头，可信度不够让人信任。

### 甜蜜点：可见数据 + 叙事娱乐

成功参考：

**ESPN底部数字条** — 体育直播时角落里实时滚动的球员数据。观众在看比赛时不读数字，但知道数字在那里，这本身就建立了可信度。

**虎扑大神分析帖** — "近5场主场战绩3W1D1L，对比客场2W1D2L，优势明显" — 数字+口语化，有数据有观点，读起来不像报告。

**Bloomberg Terminal** — 数字极其密集，但每个数字都是实时的、有来源的，**密度本身就是可信度信号**。

### 设计原则

> **不是"更学术"，而是"让数据可见"**  
> 方法论不需要解释，需要标注  
> 数据不需要展开，需要高亮  
> 历史记录不需要图表，需要小徽章

具体操作翻译：
- ❌ 不要加"关于冰狗的方法论"段落
- ✅ 要在冰狗每条发言旁边加一行小字："Poisson模型 · 进失球统计"
- ❌ 不要做准确率分析报表
- ✅ 要在冰狗的卡片上加"近10场：✓✓✗✓✓ 70%"
- ❌ 不要展示置信区间
- ✅ 要让Agent说"模拟10000次，主胜52%±4%"（这已经在做了，需要**视觉高亮**）

## 四、视觉改造方案（分级）

### 第一级：信息层级修复（改变空间分配）

**当前三列布局的问题**：3D场景是中心但不是信息中心。

**新布局方案A：数据驾驶舱布局**（激进改动）

```
┌─────────────────────────────────────────────────────────┐
│  顶栏：两队信息 + 赔率 + 阶段指示           [高60px]    │
├──────────────┬──────────────────────┬────────────────────┤
│              │   中央发言区域        │  实时数据面板       │
│  Agent头像   │  ┌────────────────┐  │  ┌──────────────┐  │
│  纵向排列    │  │ 当前发言者大卡  │  │  │ 引用的数据   │  │
│  6个小圆圈   │  │  发言内容      │  │  │  高亮显示    │  │
│  显示立场    │  │  方法论标签    │  │  │              │  │
│  和准确率    │  └────────────────┘  │  │ 概率走势图   │  │
│              │  历史发言（可折叠）  │  │ （拔河绳）   │  │
│  [150px]     │     [560px]          │  │  [370px]     │  │
├──────────────┴──────────────────────┴────────────────────┤
│  底部：比赛选择 + 控制按钮 + 状态              [高80px]  │
└─────────────────────────────────────────────────────────┘
```

3D场景降为**背景层**（全屏毛玻璃背景），不占独立区块空间。

**新布局方案B：保留3D但改变比例**（保守改动）

保持左中右三列，但调整比例：
- 左侧Agent面板：280px（从148px扩大，加入立场标签+准确率）
- 中央3D场景：560px（从984px缩减）  
- 右侧数据面板：340px（新增，显示当前会话引用的数据）

这样3D场景从77%缩到44%，释放出大量空间给有用信息。

**推荐方案B**，改动成本较低，同时能立刻解决信息密度问题。

---

### 第二级：可信度信号注入（不改布局，只加内容）

**必须加的4个可信度信号**：

**① Agent准确率徽章（最重要）**

每个Agent卡片上显示：
```
┌─────────────────┐
│ 📊 Dr.冰狗      │
│ 数据帝          │
│ ✓✓✗✓✓✓✗✓  70%  │
│ 近8场 · 上轮✓   │
│ ─ 当前: 未表态  │
└─────────────────┘
```

这是单个改动能带来最大可信度提升的优化。

**② 发言方法论标签**

每条发言附一行小字标注：
```
Dr.冰狗  [Poisson模型 · 历史进失球]  初判
"主胜52%/平28%/客20%——近5场主队进1.4球/场，
均值回归信号在第3场即将触发..."
```

**③ 数据引用实时高亮**

当Agent引用一个数字（如"1.4球/场"），对应的数据源在右侧面板高亮3秒：
```
⚡ 刚被引用
• 桑德兰近5场进球: 1.4/场
  来源: football-data.org · 实时
```

这一个改动能回答"你怎么得到的"这个问题。

**④ Evidence Board激活**

现在Evidence Board显示"0%共识"空列表，完全没用。  
修复：每条发言后自动追加Agent引用的关键数据点到"已确认"列。

---

### 第三级：概率条重设计（从静态到动态）

**当前问题**：三段蓝/绿/红，静止，没有戏剧感，深绿不可辨。

**新方案：拔河绳设计**

```
桑德兰       ████████████████▓▓▒▒░░░░░░░░░░░░░░  诺丁汉森林
              <── 主队有优势 ──● ── 客队有优势 ──>

当前预测走势: 主胜48%  平29%  客23%
绳子位置: 偏向主队方向，摆幅中等
```

核心差异：
- 一条绳而非三段条
- 绳子有**物理弹性**（每次发言后抖动再稳定）
- 中线是平局点，绳偏左=主队优势，偏右=客队优势
- 用户的预测是固定钉在绳上的锚点（全程可见）
- 每次Agent发言后绳子实时移动

这把"预测是个静态结果"变成"预测是个动态博弈"——大幅提升戏剧感。

---

### 第四级：3D场景方向（两个路线）

**路线A：放弃3D人形，改成"数据空间"美学**（激进）

- 不再是人形角色，改为6个浮动数据节点（光球/能量场）
- 每个节点代表一个Agent，节点大小 = 当前置信度
- 节点之间有连线（表示"引用了对方的论点"）
- 背景是宏大的数据流瀑布（彭博/路透风格）
- 整体美学：深蓝 + 白数字 + 橙色高亮 = 专业数据机构风

**路线B：升级3D人形，从玩具到Vtuber**（保守）

保留人形但重新设计：
- 比例：头1/身5（现在接近1/3，太短）
- 服装：有职业特征（冰狗连帽衫+大框眼镜，赌狗西装领带，月影姐月亮发饰）
- 发言时有口型动作（morphTarget或简单骨骼）
- 平时有idle动作（轻微呼吸/摇头）
- 当被crit/被选边时有明显的肢体反应

路线B成本更高，但和当前代码基础兼容。路线A改动最大但效果最一致。

---

### 第五级：色调调整

**当前**：深绿底色 + 霓虹绿金 = 电竞游戏风  
**问题**：传达"娱乐/游戏"，不传达"数据/专业"

**新方向**：深蓝底色 + 白字 + 世界杯金色点缀

参考：
- Sky Sports直播间：深蓝底色 + 白字 + 蓝金配色
- Bloomberg终端：黑底 + 橙高亮数字
- FIFA官方数字产品：深蓝 + 金色LOGO

方案：
- `--bg-deep`: `#010714` (深蓝，非深绿)
- `--accent-primary`: `#0066ff` (蓝，非绿)
- `--accent-gold`: `#c8a832` (金，保留)
- `--text-primary`: `#e8f0ff` (冷白，非暖绿)
- 仅保留少量绿色（表示"正确"/"共识"状态）

## 五、Agent对话人设的"可信度-趣味性"平衡设计原则

### 当前问题

Agent发言现在的结构：
```
[开头梗词] + [分析内容] + [catchphrase金句]
```

**可信度不够**的原因：分析内容里有数字，但用户不知道数字哪来的。

### 解决方案：三层发言结构

每条发言在UI上呈现三层（不是Agent输出三层，是前端渲染时分层显示）：

```
层1 [小字/灰色]：方法来源标签
"Poisson模型 · football-data.org近5场数据"

层2 [正文/主色]：发言内容（现在的speech）
"绷不住了，主队近5场进1.4球/失1.2球，模拟1万次主胜52%，
但警告：主队近3场超水平，均值回归信号在第58分钟触发"

层3 [高亮/金色]：catchphrase
"数据说2-1，但均值不相信这支队的手气"
```

用户看到的是一条完整发言，但视觉上已经帮他分层：
- 哪是来源（灰色，背景）
- 哪是分析（白色，主体）  
- 哪是观点（金色，高亮）

这不需要用户学习，视觉层级自然引导眼睛。

### Agent的"权威感"来自哪里

人类评论员的可信度来自：历史记录 + 专业背景 + 数据引用

AI Agent的可信度来自：
- **历史记录** → 准确率徽章（✓✓✗✓ 70%）
- **专业背景** → 方法论标签（Poisson模型 / 盘口信号）
- **数据引用** → 引用时高亮数据来源

这三个视觉信号，不需要Agent说话更"学术"，只需要**把已有信息显示出来**。

## 七、不做的事

**不做**：
- 加学术性解释段落（"关于Poisson模型的介绍..."）
- 加图表和数据报表
- 加置信区间和误差棒
- 减少弹幕梗词（这是差异化的娱乐价值，保留）

**做**：
- 让已有数据在视觉上可见
- 让已有方法论在发言旁边可见（1行）
- 让已有历史记录在Agent卡片上可见（图标串）

**核心原则**：Bloomberg风格的数据可见性 + 虎扑风格的表达方式 = 可信的娱乐预测
