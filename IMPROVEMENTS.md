## ██ 一、当前版本（v2.1）批判性评估

### 1.1 做了什么

v2.0 加入了真实数据接入（football-data / odds-api）、比分预测取代胜负三选、加权历史准确率。  
v2.1 在此基础上：
- 信息不对称注入（攻方独占数据注入 directive）
- 辩论动态停止（共识≥75% / 连续2轮无转向 / 5轮硬上限）
- 部分立场更新（confidence 微调 ±0.1）
- `extractAgentDataValues()` 直接把字段值注入 directive，禁止 LLM 编造数字
- `buildDebateDirective()` 暴露对方的 structured 判断（pick/score/confidence/keyFactor），而非只给文字片段
- 初判阅读时长按 speech 字数自适应（每字 35ms，3-6s）

### 1.2 根本问题：没解决"有迹可循"

即使数据注入到位，每个 agent 的发言仍然是**可预测模板的变体**：

| Agent | 每次都会说的 | 永远不会说的 |
|-------|------------|------------|
| 冰狗 | "绷不住了，场均X球，推断Y-Z" | "我不确定，数据在这个问题上是噪声" |
| 赌狗 | "钱不说谎，隐含比分X，梭哈" | "市场这次可能集体犯错" |
| 月影姐 | "玄学数据说冷门，各位快反向" | "我其实这次支持热门，星象很明确" |
| 老球迷 | "那年我亲眼所见，历史规律..." | "这场没有历史参照，我真的不知道" |
| 碎碎念 | "我注意到一个细节，球员心理..." | "这支队的心理状态和数据一致，冰狗说得对" |

**问题本质**：角色人设锁死了每个 agent 的可能空间。他们的"论点"是角色标签，而不是从实际数据推导出的真实推断。

### 1.3 评价体系的缺陷

当前隐式评价体系只关注：
- 预测准确率（H/D/A + 比分命中等级）
- 置信度校准
- 数据引用率

**这是预测模型的指标，不是娱乐节目的指标。**

好的预测娱乐节目（如 ESPN 赛前分析、虎扑争议帖）的真正价值是：
1. 我看完后对这场比赛有了**新的角度**，而不只是一个数字
2. 两个人争论时有**实质分歧**，不是风格分歧
3. 过程中有**转折点**——某个信息出现改变了判断方向
4. 每场比赛的议会**和上一场感觉不同**

### 1.4 具体技术问题

| 问题 | 位置 | 说明 |
|------|------|------|
| AGENT_DIRECTIVE 模板化太强 | agents.mjs | 三步推导法让 LLM 按固定套路走，输出结构化但缺乏意外 |
| 辩论对线是两个 agent，其余三人沉默 | agents.mjs | 4个 agent 只看不说，错失信息 |
| vote 阶段没有利用辩论内容 | agents.mjs | 投票只是重写初判，不是辩论结论的延伸 |
| 月影姐没有内在逻辑 | agents.mjs | 纯"玄学+毒奶"，任何数据都得出"快反向"，观众很快识破 |
| 议长发言质量低 | agents.mjs | 只是推进和煽情，不真正影响走向 |
| 停止条件阈值过高 | agents.mjs | 共识≥75% 几乎不可能在5轮内触发 |

## ██ 二、新评价体系（v3.0 设计基础）

### 核心转变：从"预测准确率"到"值得看第二遍"

**四个维度：**

#### 维度1：信息增量（Information Gain）
- 每个 agent 是否揭示了一个观众自己找不到的角度？
- 衡量：发言中是否有"这个角度是我没想到的" - 非显而易见的推断
- 差的案例：冰狗说"主队近5场3胜"（观众自己会看）
- 好的案例：冰狗说"主队近5场当客队胜率83%但主场只有40%，今天是主场"（反直觉的数据结论）

#### 维度2：立场多样性（Position Diversity）  
- 5个 agent 的判断分布有多大差异？
- 衡量：winner 分布的熵值。全押一队 = 0（boring），三方都有 = 高（interesting）
- 目标：让月影姐有时押热门、让冰狗有时承认数据不够、让老球迷有时说"历史对这场没参考价值"

#### 维度3：因果质量（Causal Argument Quality）
- Agent 是在做"因为X所以Y"的推断，还是"当A时通常B"的相关性引用？
- 差的案例："赔率低，所以可能爆冷"（相关性）
- 好的案例："赔率低是因为市场认为主力缺阵，但新闻显示已经复出——所以市场定价是错的"（因果链）

#### 维度4：叙事张力（Narrative Tension）
- 会话内是否有一个问题，在结束前没有答案？
- 是否存在一个 pivot moment：某个信息出现后让之前的判断感觉需要修正？
- 结局是否感觉"earn't"而非随机？

### 评分应用

| 每场议会结束后 | 问题 |
|--------------|------|
| 信息增量 | "你学到了一个新角度？(Y/N)" |
| 立场多样性 | 计算 5 agent 的 winner 分布熵 |
| 因果质量 | 检测 speech 中是否包含因果词（"因为/由于/导致/所以"）而非相关词（"通常/一般/历史上"） |
| 叙事张力 | 是否有 pivot（stance change），是否有 debate_stop (consensus/stalemate) |

## ██ 三、v3.0 Agent 重设计

**设计原则**：每个 agent 要有（1）方法论（不只是人设）、（2）系统性盲点（不只是风格）、（3）与特定对手的张力（不只是"对线"）

---

### 🔵 冰狗 v3 — "均值回归派"

**方法论**：Poisson 过程思维。认为进球是随机独立事件，热门的主队不会因为"状态好"多进球，状态好只是小样本噪声。  
**核心主张**：过去4场进球数 / 对方防守失球率 → 期望进球区间。比分就在这个区间里。  
**系统性盲点**：完全不相信"状态"和"心理"是真实变量。当一支队伍连赢4场，他会说"这是均值回归的时候了"，但实际上有时候状态真的是真实的。  
**张力**：vs 碎碎念（"心理是真实变量"）、vs 赌狗（"市场比模型更快"）  
**何时说意外的话**：当数据显示主队统计更好但他仍然预测平局甚至客队赢——因为主队此前几场"超水平发挥"。  
**新规则**：禁止说"数据支持某队"——必须给出 Poisson 推断下的 P(home) P(draw) P(away) 三个数字。

---

### 🟢 赌狗 v3 — "信息不对称猎手"

**方法论**：盘口不只是预测，是信息聚合。他区分"散户资金"和"职业资金"，当两者方向相反，职业资金大概率是对的。  
**核心主张**：找盘口上"被错误定价的一边"——散户把赔率压到了错误方向。  
**系统性盲点**：有时候过度解读异动。0.05 的赔率变动他会说"这是内幕"，其实可能只是流动性。  
**张力**：vs 冰狗（"市场是模型的对手，而不是验证"）、vs 月影姐（"她说反向，我看市场谁在反向"）  
**何时说意外的话**：当他说"这次市场可能集体犯错了，散户和职业都在错误一侧"——真正的逆向，需要理由。  
**新规则**：必须区分"公众盘"和"专业盘"方向。有时两者一致（顺势），有时对立（这才是他最有价值的时候）。

---

### 🟡 老球迷 v3 — "情景比较主义者"

**方法论**：不只是 H2H 记录，而是找"最相似的历史情景"——不只比对手，而是比赛时机、赛季节点、球队状态模式。  
**核心主张**：当前这场比赛最像历史上的哪一场？那场怎么结束的？  
**系统性盲点**：确认偏误——他会找到支持他直觉的历史案例，而忽略反例。冰狗最喜欢戳他的这个点（"你说1998年，但你没提2003年和2007年正好相反"）。  
**张力**：vs 冰狗（"n=3 不是样本，你在讲故事"）、vs 月影姐（"她的非因果解释和他的历史类比有时殊途同归，让人困惑"）  
**何时说意外的话**：当他说"这次历史没有参照，我真的看不出规律" —— 承认自己的方法论在这场比赛上失效了。  
**新规则**：必须给出具体的历史比赛（年份+对阵），并承认反例的存在。不能只选有利于自己结论的案例。

---

### 🔴 碎碎念 v3 — "行为信号分析师"

**方法论**：从可观察的公开信号重建球员心理状态——赛后采访语气、训练场视频、社媒动态、伤病/缺阵模式。不只是"士气好/坏"，而是具体球员的具体心理状态。  
**核心主张**：找到一个关键球员——这场比赛的结果高度依赖这个人的心理状态。  
**系统性盲点**：过度拟人化。把一个球员拍的一张皱眉照片解读成"比赛动机不足"，有时是过度解读。  
**张力**：vs 冰狗（"你的变量无法量化，不算证据"）、vs 赌狗（"个体心理和市场聚合信息的层级之争"）  
**何时说意外的话**：当她说"这场的关键球员心理状态出奇好，我同意冰狗的数据结论，但原因不同"——两个对立角色达成相同结论，但理由完全不同，这是最有趣的时刻。  
**新规则**：必须点名一个具体球员，并给出可观察的信号来源（"赛后采访里X说了Y"）。不能只说"主队士气高"。

---

### 🟣 月影姐 v3 — "反叙事猎人"（重大重设计）

**这是变化最大的 agent。**

**真实方法论**（被玄学包装的内核）：她跟踪"公众叙事过热"的时机。当所有人都说某队一定赢、媒体报道全部偏向某边、赔率已经完全反映这个叙事——这时叙事往往会以某种方式失败。她的"玄学"是表演，她的真正逻辑是反叙事套利。  
**核心主张**：找到当前最强的公众叙事，然后找出它脆弱的地方。  
**系统性盲点**：为了逆向而逆向。有时叙事是准确的，但她仍然找冷门，因为那是她的"品牌"。  
**张力**：vs 赌狗（"他也是逆向的，但他看资金；她看叙事——当两者一致，是最强的逆向信号"）、vs 冰狗（"数据也是一种叙事"）  
**何时说意外的话**：当她说"这次我支持热门，因为冷门叙事本身已经过热了" —— 逆向的逆向，最高层次的不可预测。  
**新规则**：必须指出当前"最强的错误叙事"是什么，然后说明为什么它脆弱。玄学只是修辞，内核必须是叙事分析。

---

### ⚖️ 议长 v3 — "有偏见的主持人"

**方法论**：持有自己的判断，但选择性暴露。不是中立主持人，是有立场的provocateur。  
**新能力**：  
- 在辩论中可以"钦点"某个 agent 的论点为"本轮最强"（影响后续 agent 的方向）  
- 可以提出其他人都没提到的角度（从全局数据里）  
- vote 阶段前可以做一个"议长内部预测"，但不公开，最后揭晓  
**张力**：选择支持哪边时有明显倾向（这轮喜欢哪个 agent），但表面上是公正主持  
**新规则**：每次对线后必须说"这轮 [X] 的论点更强，因为 [具体理由]"，而不是"两边都有道理"。

## ██ 四、v3.0 Battle 形式重设计

### 核心原则
- 每场 battle 的叙事弧要不同（不同比赛产生不同走向）
- 停止条件由内容驱动（有人转向 = 真实转折），而非时钟驱动
- 非对线 agent 也要参与，而不是沉默

### 新流程："三庭制"

```
═══════════════════════════════════════════════════
  法庭一：独家证词（Initial Evidence）
═══════════════════════════════════════════════════
  各 agent 提交「一条最重要的数据/判断」
  规则：只有这一条，不能给预测
  目标：暴露信息地图，让信息差可见
  结束：所有 agent 提交完毕 → 议长宣布最大信息分歧

═══════════════════════════════════════════════════
  法庭二：交叉质询（Cross Examination）
═══════════════════════════════════════════════════
  议长安排 2 组质询（不总是同两个人）：
    - 第1组：信息分歧最大的两个 agent
    - 第2组：议长认为还有角度没被覆盖的配对
  每组质询：2来回（不是4）
  质询目标：攻击的是对方的「方法论」，不是结论
  第三个 agent 可以在任一组质询中「插嘴」（有独占信息时）
  停止条件：有人转向 OR 两组完成

═══════════════════════════════════════════════════
  法庭三：终判场景（Final Scene）
═══════════════════════════════════════════════════
  每个 agent 锁定一个具体场景（而非 H/D/A）：
  - 比赛第 X 分钟
  - 球员名 + 具体动作
  - 这个画面如何出现（推理路径，1句话）
  Vote 是：你相信哪个场景会发生？
  最终输出：场景频率最高的比分为议会综合预测
```

### "立场飘移"机制（代替固定轮数）

辩论没有固定轮数。停止条件改为：

| 触发条件 | 立即动作 |
|---------|---------|
| 任意 agent 主动修改预测 | 【转折点】特效 + 让修改者解释触发原因 |
| 全部 agent 与同一 pick 一致 | 【提前共识】直接跳到法庭三 |
| 法庭二完成 2 组质询且无转向 | 议长宣布「立场固化」+ 进入法庭三 |
| 5轮硬上限 | 强制进入法庭三 |

### 议长的"钦点"权

法庭二每轮结束后，议长说：  
「[X] 的论点在这轮胜出——因为 [1句具体理由]，[Y] 的方法论在这个问题上有一个漏洞。」  
这让每轮有明确的"赢家"，创造叙事张力。

**日期**：2026-04-24  
**主题**：将预测体验从"猜胜负"升级为"猜比分"；完善数据质量、协同机制、记忆闭环。

### Breaking Changes
- 用户参与：三选一（H/D/A） → 比分选择器（0-5 × 0-5）
- `structured.score` 字段从可选变为 initial/vote 阶段必填
- `buildMatchData()` 新增 `homeTeamId`/`awayTeamId`/`fixtureId` 字段

---

### [数据层] dataFetcher.mjs

| 类型 | 内容 |
|------|------|
| NEW | `calcGoalStats(matches, teamId)` — 计算队伍场均进/失球数 |
| NEW | `getH2HScoreDistribution(h2hMatches, homeTeamId)` — 历史交锋比分频率分布 |
| NEW | `inferScoreFromOdds(homeOdds, drawOdds, awayOdds)` — 赔率隐含比分区间推断 |
| MOD | `buildMatchData()` — 保存 `homeTeamId`/`awayTeamId`/`fixtureId`；briefing新增 `avgHomeGoals`/`avgAwayGoals`/`h2hScoreFreq`/`impliedScore` |

**设计依据**：
- `avgGoals`：stat agent 通过场均进失球交叉计算最可能比分区间（简化版 Dixon-Coles）
- `h2hScoreFreq`：history agent 有历史比分分布，而非只有胜负统计
- `impliedScore`：gambler agent 从真实赔率推断隐含比分，比空字符串"无"更有说服力

---

### [记忆层] memory.mjs

| 类型 | 内容 |
|------|------|
| NEW | `calcScoreHitLevel(predicted, actual)` — 返回命中等级：`perfect/precise/valid/close/miss` |
| MOD | `saveEpisode()` — 新增 `predictedScores`/`actualScore`/`councilScore` 字段 |

**命中等级定义**：
- `perfect`：精确比分完全吻合
- `precise`：方向正确 + 总进球数误差 ≤1
- `valid`：方向正确（胜/负/平）
- `close`：方向错但总进球数误差 ≤1
- `miss`：其他（方向错且进球数也偏差大）

### [服务层] server.mjs

| 类型 | 内容 |
|------|------|
| NEW | `enrichMatchBriefing(match)` — 延迟加载真实 homeForm/awayForm/H2H，补充 briefing 数据字段 |
| MOD | `POST /api/match/:id/prepare` — 并行执行 enrichMatchBriefing + generateSeeds |
| MOD | `GET /api/run` — 加载 agentProfiles 并传入 runCouncil options |
| NEW | `POST /api/result` — 赛后录入实际比分，触发 updateAgentAccuracy + saveEpisode |

**设计依据**：

### [Agent层] agents.mjs

| 类型 | 内容 |
|------|------|
| MOD | `AGENT_DATA_ACCESS` — stat 加 `avgHomeGoals/avgAwayGoals`；gambler 加 `impliedScore`；history 加 `h2hScoreFreq` |

- 准确率加权：`agentProfiles.correct/total` 影响最终综合概率计算，历史准的 agent 权重更高
