## 二、问题诊断

### 2.1 根本问题：与真实评论席结构完全相同

这是所有问题的根源。

典型赛前评论席：专家拿同样的公开信息 → 各自分析 → 主持人串场 → 投票预测  
本项目 v0.3：Agent 拿同样的公开信息 → 各自发言 → 议长串场 → 投票预测

结构完全相同。换皮不解决本质问题，需要从协作模式和内容框架上真正改变。

### 2.2 具体问题清单（v0.3）

**协作层：**
- Debate 固定 1v1，其余 3 人沉默，6个 Agent 信息完全对称
- Vote 阶段不引用辩论内容，等于重写初判
- moderator 只出现 2 次，存在感弱
- Agent 间没有真正信息流动，初判后互不影响

**呈现层：**
- 每个阶段视觉完全相同，无阶段感
- 用户是纯旁观者，零参与
- 每次议会是孤立事件，没有持续叙事
- 底部历史条可读性差，英雄卡时长固定

## 三、方案 A：赛季制预测评论席（评论席进化版）

> 保留评论席形式，从三个维度升级：核心视觉、用户角色、Agent 协作模式

### 3.1 核心改变：概率条成为主视觉

**原则：让用户盯着一条线，而不是一段文字。**

每次 Agent 发言结束后，根据所有已表态 Agent 的 `structured.winner + confidence` 重新计算三方概率，概率条平滑滑动到新位置。辩论阶段两个 Agent 把条拉来拉去，投票阶段每人锁定一段，最终全部凝固。

**视觉形态**：3D 场景中央放置一个物理天平（或拔河绳），Agent 发言后天平实际偏转，而非平面进度条。比赛前真实赔率作为初始基准线打在条上，观众能看到 Agent 们在把共识往哪个方向推、推了多远。

**辩论阶段的震荡效果**：冰狗发言 → 条左移 → 月影姐反驳 → 条右拉 → 每一轮旁边显示"本轮移动量 +8%"，让"谁赢了这一轮"有量化依据。

**投票锁定动效**：每个 Agent 完成投票后，对应区段"凝固"（颜色变实、边缘出现锁定线）。5票全部锁定后，最终数字以打字机效果逐位显出。

### 3.2 用户角色：轻量参与

**参与方式**：议会开始前，用户做一个预测（主队胜/平/客队胜，可选比分）。用户的预测作为**标记点**固定在概率条上，全程可见。

用户体验节拍：
1. 选比赛 → 做预测（30秒以内）
2. 看议会：概率条在移动，用户的标记点是固定的——"条在往我这边靠还是离我而去？"
3. 结束后：你的预测 vs 5个 Agent 的最终立场对比——"冰狗和赌狗和你一致，月影姐押的是另一边"
4. 比赛结束后：回来看谁说对了

**跨场次积累（本地存储）**：
- 用户自己的预测准确率跨场次保留
- 对比每个 Agent 的历史准确率
- 若干场次后出现："你的准确率 58%，和月影姐差不多，比冰狗低 14%"
- 用户开始思考：下一场要不要跟冰狗一致

这不改变任何 Agent 行为，只是给用户一个"在看比赛之前有自己的立场"的钩子。

### 3.3 Agent 协作模式：三项升级

#### 升级 A：信息不对称（最高戏剧性）

**核心**：每个 Agent 开局持有一张其他人不知道的独家情报，辩论变成**择机揭牌的博弈**。

各 Agent 的情报类型：

| Agent | 独家情报类型 | 示例 |
|-------|------------|------|
| 赌狗 | 盘口异动 | "开赛前3小时，大资金把主队赔率从1.8压到2.3" |
| 碎碎念 | 球员私状态 | "10号球员赛前48小时与主教练发生争执" |
| 历史区 | 隐藏历史 | "1998年几乎相同对阵以3-0冷门收场" |
| 冰狗 | 训练隐性数据 | "客队最近3场冲刺距离下降15%，隐性疲劳" |
| 月影姐 | 超自然征兆 | "今晚月蚀，历史数据客场球队月蚀夜胜率73%" |

**规则**：初判阶段禁止揭牌，只用公开信息。辩论阶段可选择揭牌——揭牌一次性，揭了所有 Agent 立知，对方当场回应。

**Prompt 实现**：
```
初判 directive 追加：
"你持有独家情报：[agentSeed.exclusiveIntel]。本阶段禁止透露。"

辩论 directive 追加：
"你现在可以选择亮出情报牌（不可逆，其他Agent将立刻知晓）。
如果其他 Agent 亮牌，必须认真回应这条新信息对你判断的影响。"
```

**揭牌时刻视觉**：Agent 卡片出现"情报解密"动效，概率条大幅震动，3D 场景对应几何体爆发光晕。这是整场议会最大的视觉事件。

#### 升级 B：动态角色分配

**核心**：根据比赛特点，开局强制分配 Agent 的辩论立场，投票阶段才亮出真心话。

```javascript
function assignDebateRoles(match) {
  const oddsGap = Math.abs(match.odds.away - match.odds.home);
  if (oddsGap > 1.5) {
    // 强弱悬殊：2人强制扮演"冷门捍卫者"，2人扮演"热门卫士"，1人自由
    return { upset: pickTwo(experts), favorite: pickTwo(experts), free: [remaining] };
  } else {
    // 势均力敌：2人主队阵营，2人客队阵营，1人"墙头草"
    return { home: pickTwo(experts), away: pickTwo(experts), swing: [remaining] };
  }
}
```

**墙头草角色**：势均力敌比赛里，有一个 Agent 被双方争抢说服。它的最终票是被真正说服的结果，还是利益算计，观众不知道。

**投票阶段翻牌**：每个 Agent 回答两个问题：
1. 我的分配立场是什么，我是否真的相信了它？
2. 如果没有，真实判断是什么，为什么和分配立场不同？

一个 Agent 被迫捍卫某立场3轮后发现自己真的被说服了——这个时刻极其好看。

**视觉**：Agent 卡片分配阶段显示"角色标签"，投票后翻牌显示真实预测，物理上换边的 Agent 在 3D 场景中重新排位。

#### 升级 C：实时模拟证据（最新颖）

**核心**：Agent 不给意见，给**模型输出**。辩论的焦点从"你觉得谁赢"变成"你的模型参数设错了"。

各 Agent 运行不同模型：

| Agent | 模型类型 | 示例输出 |
|-------|---------|---------|
| 冰狗 | 蒙特卡洛模拟（10000次） | 主胜63% / 平22% / 客胜15%，置信区间±4% |
| 赌狗 | 市场效率模型 | 修正主场偏差后：主胜58% / 平24% / 客胜18% |
| 历史区 | 历史相似度匹配 | 7场最相似比赛：4主胜2平1客 |
| 月影姐 | 贝叶斯先验+主观调整 | 先验50/30/20，引入修正后：35/30/35 |
| 碎碎念 | 心理状态乘数 | 10号心理状态60%，影响进攻产出-12% |

**参数战示例**：
```
冰狗："蒙特卡洛10000次，主队胜率63%。"
赌狗："你的主场优势系数用了1.15，这场是中立场地，应该是1.0。"
冰狗（接受挑战）："修正后，主队胜率降至57%。"  ← 概率条实时移动
月影姐："57%也好，63%也好，今晚月相不利于主队进攻型打法。"
冰狗："月相不是模型参数。"
月影姐："贝叶斯先验可以是任何东西。"
```

被挑战后当场修正模型输出、概率条跟着跳——这是"被数据说服"而非"被话说服"，质感完全不同。

**工具调用新增字段**：
```javascript
simulationOutput: {
  modelType: "monte_carlo" | "market_model" | "historical" | "bayesian" | "psychological",
  homeWin: number,
  draw: number,
  awayWin: number,
  keyAssumptions: string[], // 可被其他 Agent 挑战的参数列表
  confidenceInterval: [number, number]
}
```

**3D 场景**：每个 Agent 激活时，中央区域显示该模型的概率分布形状（钟形曲线或散点）。5个模型叠加时，形状互相重合或偏离。投票结束时，5个形状合并为一个最终分布。

### 3.4 赛季框架：跨场次叙事

**Agent 历史战绩**：每个 Agent 有公开的跨场次准确率记录，在其卡片上常驻显示：
```
数据帝·冰狗
近10场: ✓✓✗✓✓✗✓✓✗✓
准确率: 72%  当前连胜: 3场
```

**跨场次互怼**：月影姐上场预测错了，冰狗这场开口就能提——"上次星象带偏了所有人，今天数据怎么说才算数。" 这不是固定台词，是把上场结果注入这场的 prompt。

**议长转型**：不再只是串场主播，而是持有各 Agent 历史档案的"赛季主持人"。每次开场介绍当前积分状况："冰狗本赛季领跑，连胜3场；月影姐刚从低谷中回来……"

**用户与 Agent 的竞争**：用户的准确率和所有 Agent 的准确率放在同一个排行榜上，产生"能不能超过冰狗"的长期目标。

### 3.5 优化后的议会流程

```
赛季简报（议长）
  └── 当前各 Agent 战绩，上场结果回顾（≤40字）

Phase 1: Opening（1次）
  └── 议长开场，本场比赛背景，情绪设定

Phase 2: Initial（5次）
  ├── 每人运行各自模型，输出 simulationOutput
  ├── 情报保密，只用公开信息
  └── 黑板机制：每次发言后提取 keyInsight 注入后续 Agent

Phase 3: Debate（三人干涉，共7次）
  ├── pickConflictTriple()：核心对立2人 + 第三介入者
  ├── 议长宣布冲突："最大模型分歧——[A] vs [B]，参数差异 X"
  ├── A 攻击 B 的模型参数
  ├── B 反击 A 的模型假设
  ├── 议长引入 C："[C] 的模型显示第三路径"
  ├── C 介入（可支持A/B/独立路径），**此时可揭牌**
  ├── A 对 C 的回应
  ├── B 对 C 的回应
  └── 议长总结（可轻微偏心，≤20字）

Phase 4: Vote（5次，有记忆的锁定）
  ├── 每人必须：①引用干涉中最影响自己的发言 ②申报是否触发重新校准
  ├── 立场漂移机制：被怼次数多的 Agent 有概率翻盘（prompt 注入空间）
  └── 每票锁定时，概率条对应区段凝固

Phase 5: 揭晓
  └── 议长播报最终结果，对比各 Agent 和用户的预测，更新赛季积分
```

### 3.6 moderator 角色重设计

moderator 出现 5 次，每次角色明确：

| 出现时机 | 角色 | 发言长度 |
|---------|------|---------|
| 赛季简报 | 战绩播报员 | ≤40字 |
| Phase 1 开始前 | 开场造势 | ≤30字 |
| Phase 2 → Phase 3 转场 | 冲突宣布（带模型参数差值） | ≤30字 |
| 每轮 Debate 后 | 即时裁判（这一轮谁的参数被挑穿了）| ≤20字 |
| Phase 4 前 | 煽动锁定（"最后5票，谁的模型撑到最后"）| ≤20字 |
| 揭晓 | 结果播报+积分更新 | ≤50字 |

## 四、方案 B：预言侦探局（全新框架）

> 与评论席完全无关。这是一个以**侦探调查**为核心叙事的预测体验。

### 4.1 概念

**评论席的叙事弧**：分析 → 辩论 → 投票（平行视角汇聚）  
**侦探局的叙事弧**：接案 → 取证 → 推理 → 突破 → 立案（从混沌到清晰的单线推进）

在这个框架里，比赛结果是**一起还未发生的案件**——某件事将在90分钟后发生，调查员们需要在开球之前找出答案。

用户不是在看"专家讨论"，而是在看**一场调查如何一步步逼近真相**。

### 4.2 六个调查员身份

同样的 6 个 Agent，在侦探局框架下各有专属职能：

| Agent | 侦探职务 | 调查手段 | 说话方式 |
|-------|---------|---------|---------|
| `stat` 冰狗 | 法证分析员 | 数据建模、概率重建 | "现场数据显示……" |
| `mystic` 月影姐 | 行为侧写师 | 心理+模式分析 | "凶手的行为特征指向……" |
| `history` 历史区 | 档案专家 | 历史案例比对 | "参照1998年同类案例……" |
| `gambler` 赌狗 | 资金追踪员 | 市场/盘口情报 | "资金流向显示内部知情者……" |
| `psych` 碎碎念 | 心理审讯官 | 球员心理状态解读 | "嫌疑人的心理状态表明……" |
| `moderator` 议长 | 主任侦探 | 不直接调查，主持推理会议 | "目前掌握的证据……" |

"案件"的要素映射：
- 案件 = 比赛结果（某队进球/胜利）
- 嫌疑人/当事人 = 关键球员
- 犯罪动机 = 球队/球员的胜利驱动
- 证据 = 数据/历史/心理/市场信号
- 目击证人 = 历史上的类似比赛

### 4.3 调查流程

```
接案简报（主任侦探）
  └── 案情概述：两队即将交手，结果未知，调查小组启动

Phase 1: 独立取证（5次）
  ├── 每位调查员独立勘察，形成初步证据链
  ├── 输出：evidence（证据描述）+ suspect（关键球员）+ initialTheory（初步理论）
  └── 信息不对称：每人持有独家线索（同方案A的揭牌机制）

Phase 2: 推理会议（三人交叉质证）
  ├── 主任侦探：宣布最大矛盾点——"法证和资金追踪的结论无法同时成立"
  ├── 两个核心冲突调查员互相质疑对方的证据链
  ├── 第三调查员提出独立路径
  ├── **突破时刻**：某调查员揭出独家线索 → 整个推理方向改变
  └── 主任侦探：阶段性总结（偏心，制造悬念）

Phase 3: 立案投票（5次）
  ├── 每位调查员提交"嫌疑人 + 犯罪理论"（即最终预测）
  ├── 必须引用推理会议中改变自己判断的证据
  └── 允许推翻初步理论（"新证据迫使我重新评估"）

结案报告（主任侦探）
  └── 综合所有调查员的立案理论，输出"最可信犯罪重建"（即最终预测场景）
```

### 4.4 语言体系

侦探框架下，整个界面语言完全替换：

| 评论席语言 | 侦探局语言 |
|-----------|-----------|
| 召开议会 | 启动调查 |
| 初判 | 独立取证 |
| 辩论 | 交叉质证 |
| 投票 | 立案 |
| 发言历史 | 案件档案 |
| 胜率 | 可信度 |
| 预测结果 | 结案报告 |
| Agent 卡片 | 调查员档案 |

### 4.5 视觉体系

**整体美学**：不是电竞 HUD，也不是游戏界面——是**侦探档案室**。

核心视觉元素：
- **证据板（Evidence Board）**：替代 3D 议事厅。中央是一个动态的证据连线板——球员照片、数据节点、历史案例卡片，用彩色线条连接。随着调查进行，节点增加，线条变化，直到最终形成清晰的"结案路径"
- **调查员档案卡**：替代 Agent 卡片。每张档案卡有徽章（职务）、结案率（历史准确率）、当前"可信度等级"
- **案件时间线**：替代概率条。不是胜率，而是"当前最可信结果的确定度"——确定度随证据积累上升，矛盾证据出现时下降
- **突破时刻特效**：某调查员揭出独家线索时，证据板上相关节点突然点亮，线条重新连接，整个板面重组

**与方案A的视觉差异对比**：

| 元素 | 方案A（评论席） | 方案B（侦探局） |
|------|--------------|--------------|
| 中央视觉 | 天平/概率条 | 证据连线板 |
| Agent 展示 | 卡片（名字+胜率）| 档案（徽章+结案率）|
| 发言方式 | 评论弹出气泡 | 调查报告文件打开 |
| 3D 场景 | 几何体圆阵 | 三维证据空间（节点网络）|
| 关键时刻 | 参数被挑穿时条移动 | 突破时全板面重组 |

### 4.6 侦探局特有机制

**证据链强度（替代置信度）**：每条证据有"链长"（支持这个结论的证据节点数量）。链越长，可信度越高。立案投票时，链短的调查员会被要求"补充证据"。

**证据矛盾指数**：当两个调查员的证据链指向不同结论，系统显示"矛盾指数"——矛盾越高，最终结案报告的不确定性越大。用户能直观看到"这案子好不好破"。

**结案率排行榜**：和方案 A 的赛季积分类似，但语言是"XX 调查员本赛季结案率 72%，连续3案精准破案"。每场新比赛是新的案件。

## 五、两方案对比

| 维度 | 方案A：赛季制评论席 | 方案B：预言侦探局 |
|------|-----------------|---------------|
| 核心叙事 | 赛季积分竞技 | 单案调查推进 |

**不互斥**：两个方案共用同一套 Agent 协作升级（信息不对称+动态角色+实时模拟），只是在叙事框架和视觉上走不同路线。可以先实现方案A（成本低），方案B作为长期目标。
